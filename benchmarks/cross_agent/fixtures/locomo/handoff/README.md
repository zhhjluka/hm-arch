# LoCoMo real-CLI handoff artifacts (MEM-78)

Tracked production cross-agent memory comparison artifacts. Generated by:

```bash
scripts/run_locomo_matrix_handoff.sh
```

## Contents

| File / directory | Description |
|------------------|-------------|
| `matrix_summary_real.json` | Full 4×5 matrix with metrics, provenance, and cell status |
| `matrix_summary.json` | Pointer to the active report |
| `locomo-<agent>-<backend>-*/` | Per-run `summary.json`, `queries.jsonl`, `invocations.jsonl` |

## Provider authentication

Real CLI invocations require provider credentials:

- **Codex**: `OPENAI_API_KEY` or `codex login`
- **Claude Code**: `ANTHROPIC_API_KEY` or `claude login`
- **Hermes**: `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`, or `hermes model`

The pilot run in this directory used production CLI binaries with `cli_mode: real`
but provider auth was unavailable in the cloud agent environment. All 18 queries
failed with authentication errors (401 / not logged in). Re-run locally with
credentials for meaningful accuracy metrics:

```bash
export OPENAI_API_KEY=...
export ANTHROPIC_API_KEY=...
export OPENROUTER_API_KEY=...   # or configure hermes model
LOCOMO_HANDOFF_MAX_QUERIES=50 ./scripts/run_locomo_matrix_handoff.sh
```
